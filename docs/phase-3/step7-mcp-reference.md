# Step 7 — MCP Schema Reference

Load this at Step 7 (Build in TideLearn) before making any MCP tool calls.
Source of truth: Zod schemas in `mcp/src/lib/types.ts`. When in doubt, check the schema.

---

## Content block schemas — `add_block`

Omit `id` on all blocks and sub-items — auto-generated by `injectSubItemIds`.
Text fields accept HTML. Position is 1-based and optional (omit to append).

### Simple blocks

| Block | Fields (all required unless marked ?) |
|---|---|
| `heading` | `text: string` |
| `text` | `text: string` (HTML) |
| `divider` | *(no fields)* |
| `toc` | *(no fields)* |
| `image` | `src: string`, `alt: string` |
| `video` | `url: string` |
| `audio` | `src: string`, `title?: string` |
| `code` | `language: string`, `code: string` |
| `quote` | `text: string`, `cite?: string` |
| `document` | `src: string`, `fileType: "pdf"\|"docx"\|"xlsx"\|"pptx"`, `title?: string` |
| `button` | `label: string`, `url: string`, `variant: "primary"\|"secondary"\|"outline"`, `openInNewTab: boolean` |
| `embed` | `url: string`, `title: string`, `height: number` (100–2000) |

### Callout

```json
{
  "type": "callout",
  "variant": "info" | "success" | "warning" | "danger",  // REQUIRED
  "title": "optional title",
  "text": "callout body (HTML)"                           // REQUIRED
}
```

⚠️ `variant` is required — omitting it fails validation.
⚠️ `text` is required — omitting it fails validation.

### List

```json
{
  "type": "list",
  "style": "bulleted" | "numbered",
  "items": ["item1", "item2"]  // min 1 item, each min 1 char
}
```

### Accordion

```json
{
  "type": "accordion",
  "items": [
    { "title": "Panel title", "content": "Panel content (HTML)" }
  ]  // min 1 item. Omit id on each item.
}
```

### Tabs

```json
{
  "type": "tabs",
  "items": [
    { "label": "Tab label", "content": "Tab content (HTML)" }
  ]  // min 1 item. Field is "label" NOT "title". Omit id.
}
```

⚠️ Field is `label`, not `title`.

### Flashcard

```json
{
  "type": "flashcard",
  "front": "Question or term",
  "back": "Answer or definition",
  "hint": "optional hint"
}
```

### Timeline

```json
{
  "type": "timeline",
  "items": [
    { "date": "2024", "title": "Event title", "description": "optional" }
  ]  // min 2 items. Omit id.
}
```

### Process

```json
{
  "type": "process",
  "steps": [
    { "title": "Step title", "description": "optional" }
  ]  // min 2 steps. Omit id.
}
```

### Chart

```json
{
  "type": "chart",
  "chartType": "bar" | "line" | "pie",
  "title": "optional",
  "labels": ["A", "B", "C"],           // min 1
  "datasets": [
    { "label": "Series 1", "values": [10, 20, 30] }  // min 1 dataset, min 1 value
  ]
}
```

### Quiz (single MCQ)

```json
{
  "type": "quiz",
  "question": "Question text",
  "options": ["A", "B", "C", "D"],     // min 2 options
  "correctIndex": 0,                    // 0-based
  "showFeedback": true,                 // optional
  "feedbackMessage": "Explanation"       // optional — single field, not split correct/incorrect
}
```

⚠️ `correctIndex` is 0-based. Factory default is -1 — must be set.
⚠️ Feedback is a single `feedbackMessage` field, not `feedbackCorrect`/`feedbackIncorrect`.

### True/False

```json
{
  "type": "truefalse",
  "question": "Statement text",
  "correct": true,
  "showFeedback": true,                 // optional
  "feedbackCorrect": "Why correct",      // optional
  "feedbackIncorrect": "Why incorrect"   // optional
}
```

Note: `truefalse` uses split `feedbackCorrect`/`feedbackIncorrect`, unlike `quiz`.

### Short Answer

```json
{
  "type": "shortanswer",
  "question": "Question text",
  "answer": "expected answer",          // REQUIRED, min 1 char — cannot be empty string
  "acceptable": ["alt1", "alt2"],       // optional array of alternatives
  "caseSensitive": false,               // optional
  "trimWhitespace": true,               // optional
  "showFeedback": true,                 // optional
  "feedbackMessage": "Explanation"       // optional
}
```

⚠️ `answer` must be non-empty (`z.string().min(1)`). Empty string `""` fails validation.
⚠️ Not auto-scorable for open-ended questions — use for factual recall only.

### Multiple Response

```json
{
  "type": "multipleresponse",
  "question": "Select all that apply...",
  "options": ["A", "B", "C", "D"],     // min 2, MAX 6 options
  "correctIndices": [0, 2],             // min 2 correct, 0-based
  "showFeedback": true,                 // optional
  "feedbackMessage": "Explanation"       // optional
}
```

⚠️ Maximum 6 options — more will fail validation.
⚠️ Minimum 2 correct indices required.

### Fill in the Blank

```json
{
  "type": "fillinblank",
  "template": "The {{1}} is the {{2}} of learning.",  // use {{n}} gap markers
  "blanks": [
    { "acceptable": ["answer1", "alt1"], "caseSensitive": false }
  ],  // min 1 blank, omit id. Order matches {{n}} markers.
  "showFeedback": true  // optional
}
```

⚠️ Field is `template`, not `text`.
⚠️ No `feedbackMessage` field on content blocks — only `showFeedback`.

### Matching

```json
{
  "type": "matching",
  "prompt": "Match each term to its definition",
  "left": [{ "label": "Term 1" }, { "label": "Term 2" }],   // min 2, omit id
  "right": [{ "label": "Def 1" }, { "label": "Def 2" }],    // min 2, omit id
  "pairs": [
    { "leftIndex": 0, "rightIndex": 0 },
    { "leftIndex": 1, "rightIndex": 1 }
  ],  // min 2 pairs — use INDEX not ID
  "showFeedback": true  // optional
}
```

⚠️ `add_block` matching uses `leftIndex`/`rightIndex` (0-based integers).
⚠️ The stored schema uses `leftId`/`rightId` — IDs are injected server-side from indices.

### Sorting

```json
{
  "type": "sorting",
  "prompt": "Put these steps in order",
  "showFeedback": true,                 // REQUIRED (not optional)
  "buckets": [
    { "label": "Step 1" },
    { "label": "Step 2" },
    { "label": "Step 3" }
  ],  // min 2 buckets, omit id
  "items": [
    { "text": "Do this first", "bucketId": "<bucket-id>" }
  ]  // min 2 items, omit id
}
```

⚠️ Bucket-based architecture — NOT a simple ordered array.
⚠️ For step sequencing: create one bucket per step ("Step 1", "Step 2", etc.), one item per action, each item's `bucketId` = its correct bucket.
⚠️ `showFeedback` is REQUIRED (not optional like on other blocks).
⚠️ `bucketId` must reference a bucket — but since IDs are auto-generated, you need to either:
  - Pre-generate bucket IDs and reference them in items, OR
  - Let `injectSubItemIds` handle it (server preserves provided IDs or generates new ones)

### Hotspot

```json
{
  "type": "hotspot",
  "src": "https://image-url",
  "alt": "Image description",
  "hotspots": [
    { "x": 25, "y": 40, "label": "Feature", "description": "optional" }
  ]  // x/y are 0-100 (percentage), omit id
}
```

⚠️ Requires coordinate setup — flag for human author completion in the editor.

### Branching

```json
{
  "type": "branching",
  "prompt": "What would you do?",        // field is "prompt" NOT "scenario"
  "choices": [
    { "label": "Choice text", "content": "<p>Consequence HTML</p>" }
  ]  // min 2 choices. Fields are "label"/"content" NOT "text"/"consequence". Omit id.
}
```

⚠️ Field is `prompt`, not `scenario`.
⚠️ Choice fields are `label` and `content`, not `text` and `consequence`.

## Assessment question schemas — `add_question`

Omit `id` on all questions and sub-items — auto-generated.
All question types use `text` for the question stem (not `question`).
All question types use a single `feedback` field (not split correct/incorrect).

### MCQ (`kind: "mcq"`)

```json
{
  "kind": "mcq",
  "text": "Question stem",               // NOT "question"
  "options": ["A", "B", "C", "D"],       // exactly 4 options (tuple)
  "correctIndex": 0,                      // 0-based, max 3
  "feedback": "Explanation",              // optional, SINGLE field
  "bloomLevel": "C",                      // optional: K|C|UN|AP|AN|EV
  "source": "LT 1.1"                     // optional: learning target reference
}
```

⚠️ Content block `quiz` uses `question` + `feedbackMessage`. Assessment MCQ uses `text` + `feedback`.

### Multiple Response (`kind: "multipleresponse"`)

```json
{
  "kind": "multipleresponse",
  "text": "Select all that apply",
  "options": ["A", "B", "C", "D"],       // min 2, MAX 6
  "correctIndices": [0, 2],              // min 2 correct
  "feedback": "Explanation"               // optional
}
```

### Fill in the Blank (`kind: "fillinblank"`)

```json
{
  "kind": "fillinblank",
  "text": "The {{1}} model has {{2}} phases.",  // NOT "template"
  "blanks": [
    { "acceptable": ["ADDIE"], "caseSensitive": false }
  ],  // omit id
  "feedback": "Explanation"               // optional
}
```

⚠️ Assessment FIB uses `text`, content block FIB uses `template`.

### Matching (`kind: "matching"`)

```json
{
  "kind": "matching",
  "text": "Match each item",
  "left": [{ "label": "Term 1" }],       // min 2, omit id
  "right": [{ "label": "Def 1" }],       // min 2, omit id
  "pairs": [
    { "leftIndex": 0, "rightIndex": 0 }
  ],  // min 2 — uses INDEX (converted to IDs server-side)
  "feedback": "Explanation"               // optional
}
```

### Sorting (`kind: "sorting"`)

```json
{
  "kind": "sorting",
  "text": "Order these steps",
  "buckets": [{ "label": "Step 1" }],    // min 2, omit id
  "items": [
    { "text": "Action", "bucketIndex": 0 }  // uses bucketINDEX not bucketId
  ],  // min 2, omit id
  "feedback": "Explanation"               // optional
}
```

⚠️ Assessment sorting uses `bucketIndex` (0-based integer). Content block sorting uses `bucketId` (string).

## Content block vs assessment question — field name differences

| Concept | Content block field | Assessment question field |
|---|---|---|
| Question stem | `question` | `text` |
| FIB template | `template` | `text` |
| Feedback | `feedbackMessage` (quiz, multipleresponse, shortanswer) or `feedbackCorrect`/`feedbackIncorrect` (truefalse) | `feedback` (single field, all types) |
| Matching pairs | `leftIndex`/`rightIndex` | `leftIndex`/`rightIndex` |
| Sorting item placement | `bucketId` (string) | `bucketIndex` (0-based integer) |
| Sorting feedback | `showFeedback: boolean` (required) | *(not applicable)* |

## `update_assessment_config`

```json
{
  "course_id": "uuid",
  "lesson_id": "uuid",
  "passingScore": 80,   // optional, 0-100
  "examSize": 10         // optional, min 1
}
```

At least one field must be provided.

> **Note:** Only `passingScore` and `examSize` are supported. `timeLimit` and `maxAttempts` are not implemented.

## Common gotchas

1. **Never include `id` fields** — on blocks, sub-items (accordion items, tabs items, timeline items, process steps, sorting buckets/items, branching choices, matching left/right), or questions. All auto-generated.
2. **`schemaVersion: 1`** is required on all course JSON.
3. **`shortanswer` is NOT a supported assessment question kind** — only mcq, multipleresponse, fillinblank, matching, sorting.
4. **`multipleresponse` max 6 options** — content blocks and assessment questions both enforce this.
5. **`shortanswer` answer field must be non-empty** — `answer: ""` fails `z.string().min(1)`.
6. **Audio MIME types** — `audio/mpeg` and `audio/wav` only. NOT `audio/mp3`.
7. **Document field is `src`** — not `url`.
8. **Block operations error on assessment lessons** — use `add_question`/`update_question` instead of `add_block`/`update_block`.
9. **`correctIndex` is 0-based** — factory default is -1, must be explicitly set.
10. **Position is 1-based** — when using `add_block` with position parameter.
11. **Text fields accept HTML** — use `<p>`, `<strong>`, `<em>`, etc.
12. **Sorting is bucket-based** — not a simple ordered array. Create buckets, assign items to buckets.
